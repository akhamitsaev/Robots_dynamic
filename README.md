# Описание проекта

## Название

**Bio-inspired Multi-Agent RL / Robots Dynamic: прогнозирование динамики взаимодействующих роботов и анализ макросостояний системы**

## Аннотация проекта

Проект посвящён исследованию коллективной динамики системы взаимодействующих мобильных роботов и оценке того, как начальные условия влияют на формирование макросостояний, в частности мицеллоподобной структуры. В качестве исходных данных используются экспериментальные траектории роботов: координаты `x`, `y`, ориентация `angle`, идентификатор робота, номер временного среза и имя эксперимента. Всего в основном наборе анализировалось **307 экспериментов** и более **8.18 млн записей**, средняя длина эксперимента составляла около **591.5 временных срезов**.

Основная научная задача состоит в переходе от прогноза движения отдельных агентов к анализу поведения всей системы. Для этого в проекте реализован двухуровневый pipeline: сначала строится one-step прогноз состояния каждого робота с помощью графовой нейронной сети, затем прогнозные траектории агрегируются в системные признаки и кластеризуются по макросостояниям с помощью KMeans, а для новых траекторий выполняется присвоение метки ближайшего кластера. Такой подход позволяет исследовать не только точность координатного прогноза, но и появление, длительность и устойчивость коллективных структур.

В качестве baseline использована кинематическая модель, основанная на классических формулах движения с учётом линейных и угловых скоростей и ускорений. Основная модель — GNN с `GINEConv`, которая строит граф на каждом временном срезе: узлы соответствуют роботам, а рёбра соединяют ближайших соседей. Модель предсказывает `Δx`, `Δy`, `Δangle` для каждого робота на один шаг вперёд. По сравнению с кинематическим baseline GNN показала существенное улучшение качества one-step прогноза: снижение RMSE по `Δx` примерно на **47.7%**, по `Δy` на **45.1%**, по `Δangle` на **78.8%**.

Для анализа коллективных состояний системы реализован Macrostate KMeans pipeline. Для каждого временного среза рассчитываются 11 признаков, описывающих плотность, порядок, вращение, скорость и согласованность движения. На фактических данных обучается KMeans с **4 кластерами**, после чего обученные центроиды применяются к прогнозным и синтетическим rollout для присвоения меток кластеров. Это позволяет интерпретировать траектории в терминах макросостояний: мицелла, разреженное состояние, переходные и swarm-like режимы.

Отдельная часть проекта — synthetic macrostate study. Для исследования влияния начальных условий был реализован генератор синтетических начальных состояний и autoregressive multistep rollout на **600 шагов**. В последней серии экспериментов использовались **72 синтетических сценария**: 3 начальных радиуса области размещения роботов, 3 значения начальной скорости, 4 режима начального направления движения и 2 случайных повтора. Было показано, что начальная плотность является ключевым фактором формирования мицеллы: при радиусах **250** и **350** мицелла появлялась в **100%** экспериментов, а при радиусе **470** вероятность снижалась и сильнее зависела от режима начального движения.

Результаты проекта оформлены как воспроизводимый исследовательский pipeline с отдельными модулями для one-step прогноза, кластеризации макросостояний и присвоения меток кластеров и синтетического анализа. Код и README подготовлены для open-source публикации. Из-за большого размера файл `gnn_predictions.parquet` вынесен во внешнее хранилище Google Drive: <https://drive.google.com/file/d/1ta-cFK3d8eG9BzpDKPMLvyWPT9Jl2FXE/view?usp=sharing>.

## Проблематика

Моделирование систем взаимодействующих роботов является актуальной задачей на пересечении robotics, multi-agent systems, graph machine learning и data-driven physics. В таких системах поведение отдельного агента относительно просто описать через координаты, ориентацию и скорость, однако коллективное поведение часто приводит к emergent-структурам: роям, вращательным режимам, кластерам, разреженным состояниям и мицеллоподобным конфигурациям. Такие состояния трудно описывать только через траектории отдельных роботов.

Классические кинематические модели хорошо интерпретируемы, но ограничены тем, что не учитывают сложное взаимодействие между агентами. С другой стороны, современные нейросетевые модели могут аппроксимировать сложную динамику, но часто дают результат в виде координатного прогноза без прямой интерпретации макросостояний. Поэтому возникает разрыв между микродинамикой и макродинамикой: модель может предсказывать положение роботов, но исследователю всё ещё нужно понять, какое коллективное состояние сформировалось.

В рамках проекта рассматривается именно этот разрыв. Необходимо не только предсказывать состояние каждого робота, но и определять, как меняется состояние всей системы во времени, при каких начальных условиях появляется мицелла, насколько быстро она формируется и как долго сохраняется. Это важно для проектирования управляемых swarm-систем, анализа самоорганизации, синтетического тестирования гипотез и последующего планирования физических экспериментов.

## Постановка задачи

Цель исследования — определить, как начальные условия влияют на макросостояние системы взаимодействующих роботов, и разработать воспроизводимый pipeline для прогнозирования и анализа динамики такой системы.

В проекте решались следующие задачи:

1. Провести разведочный анализ экспериментальных данных о движении роботов.
2. Реализовать интерпретируемый кинематический baseline для one-step прогноза координат и угла.
3. Построить графовое представление системы роботов на каждом временном срезе.
4. Обучить GNN-модель для one-step прогноза `Δx`, `Δy`, `Δangle`.
5. Реализовать Macrostate KMeans pipeline для выделения коллективных состояний системы и последующего присвоения меток кластера новым траекториям.
6. Реализовать autoregressive multistep rollout, где после seed-history модель использует собственные предыдущие предсказания.
7. Создать генератор синтетических начальных условий и провести synthetic macrostate study длиной **600 шагов**.
8. Оценить влияние начального радиуса, скорости и режима движения на факт появления мицеллы, шаг её формирования и длительность существования.

Исследовательский вопрос формулируется следующим образом:

> Какие начальные условия системы роботов — плотность, скорость и режим начального движения — способствуют формированию и устойчивому существованию мицеллоподобного макросостояния?

## Описание технического решения

Техническое решение состоит из трёх связанных модулей: `StepPrediction`, `Macrostate` и `SynteticMacrostateStudy`. Вместе они образуют полный исследовательский цикл: от анализа фактических данных и one-step прогноза до multistep rollout и оценки влияния синтетических начальных условий на макросостояния.

Структура проекта: 
```text
Robots_dynamic/
├── StepPrediction/              # one-step прогноз координат и угла роботов
├── Macrostate/                  # KMeans-классификация макросостояний
└── SynteticMacrostateStudy/     # multistep + synthetic study начальных условий (объединяет решения двух предыдущих ветвей)
```

### 1. Кинематический baseline

Кинематический baseline — это физически интерпретируемая модель прогноза следующего состояния робота. Она использует текущие координаты, угол ориентации, линейную скорость, линейное ускорение, угловую скорость и угловое ускорение. Прогноз строится по классическим формулам равноускоренного движения:

```text
x_pred(t) = x(t-1) + Vx(t-1) · dt + 0.5 · ax(t-1) · dt²
y_pred(t) = y(t-1) + Vy(t-1) · dt + 0.5 · ay(t-1) · dt²
θ_pred(t) = θ(t-1) + ω(t-1) · dt + 0.5 · α(t-1) · dt²
```

Этот baseline нужен по двум причинам. Во-первых, он задаёт понятную нижнюю границу качества для data-driven модели. Во-вторых, он позволяет проверить, действительно ли GNN извлекает дополнительную информацию из взаимодействий между роботами, а не просто повторяет кинематическую экстраполяцию.

### 2. Графовое представление системы

На каждом временном срезе система роботов представляется как граф. Узлы графа — отдельные роботы. Рёбра строятся между ближайшими соседями; в основной конфигурации используется `k_neighbors = 5`. Такой подход отражает локальный характер физических взаимодействий: движение робота в большей степени зависит от ближайшего окружения, чем от всех агентов системы.

Для каждого узла рассчитывается 9 признаков:

```text
x, y, θ, vx, vy, ω, ax, ay, α
```

где `x`, `y` — координаты робота, `θ` — угол ориентации, `vx`, `vy` — линейные скорости, `ω` — угловая скорость, `ax`, `ay` — линейные ускорения, `α` — угловое ускорение.

Для каждого ребра рассчитывается 11 признаков:

```text
Δx, Δy, distance, relative_angle, Δθ, Δvx, Δvy, Δax, Δay, Δω, Δα
```

Они описывают геометрию пары роботов, расстояние между ними, относительную ориентацию и различия в динамике.

### 3. Архитектура GNN-модели

GNN решает задачу node-level regression: для каждого робота предсказываются изменения состояния на следующий шаг:

```text
Output per robot: Δx, Δy, Δθ
```

Архитектура модели:

```text
Input graph at time t
    ↓
Node features: 9 признаков
Edge features: 11 признаков
    ↓
Node encoder: Linear(9 → 128)
Edge encoder: Linear(11 → 128)
    ↓
Input node embeddings
    ↓
GINEConv #1
BatchNorm
ReLU
Dropout
    ↓
GINEConv #2
BatchNorm
ReLU
Dropout
    ↓
Predictor head
Linear(128 → 64)
ReLU
Linear(64 → 3)
    ↓
Output: Δx, Δy, Δθ
```

Модель использует два последовательных message-passing блока:

```text
GINEConv → BatchNorm → ReLU → Dropout
```

`GINEConv` выбран потому, что он поддерживает признаки рёбер (`edge_attr`). Это принципиально важно для данной задачи: относительное положение, расстояние, разность скоростей и ускорений между соседними роботами являются основными признаками взаимодействия. Первый GNN-слой позволяет роботу агрегировать информацию от ближайших соседей, второй слой расширяет локальный контекст до соседей второго порядка.

`BatchNorm` стабилизирует распределение скрытых представлений, `ReLU` добавляет нелинейность, `Dropout` снижает риск переобучения. Финальный predictor head преобразует скрытое представление каждого робота в три прогнозируемые компоненты: `Δx`, `Δy`, `Δθ`.

Обучение выполняется как one-step prediction: на вход подаётся граф в момент времени `t`, целевым значением является разность между состоянием `t+1` и состоянием `t`. Используются `MSELoss`, оптимизатор `AdamW`, регуляризация `weight_decay`, scheduler `ReduceLROnPlateau` и gradient clipping для стабилизации обучения.

### 4. Multistep autoregressive прогноз

После обучения one-step GNN используется для multistep rollout. В отличие от one-step режима, где для каждого шага используется фактическое состояние, в multistep режиме модель после начальной истории использует собственные предыдущие прогнозы:

```text
fact history: t=0..3
    ↓
predict t=4
    ↓
recompute velocities and accelerations from predicted history
    ↓
predict t=5
    ↓
repeat until N steps
```

Такой режим позволяет исследовать долгосрочную динамику системы, но одновременно является более сложным: ошибка может накапливаться, а траектория постепенно уходить от фактического распределения данных. Поэтому multistep прогноз используется не только как точный координатный прогноз, но и как генератор возможных траекторий для последующего macrostate-level анализа.

### 5. Macrostate-кластеризация

Для анализа коллективного поведения одной только ошибки координатного прогноза недостаточно. GNN предсказывает микродинамику — координаты и угол каждого отдельного робота, однако исследовательский вопрос относится к макродинамике: какое коллективное состояние принимает вся система, когда возникает мицеллоподобная структура, насколько долго она сохраняется и как это зависит от начальных условий. Поэтому поверх координатного прогноза реализована отдельная unsupervised-модель макросостояний на основе KMeans.

Важно терминологически разделять два этапа:

```text
1. Обучение KMeans на фактических данных = кластеризация временных срезов системы.
2. Применение обученной KMeans-модели к новым rollout = присвоение метки ближайшего кластера.
```

KMeans не является supervised-классификатором: в модели нет заранее размеченных экспертами классов. Алгоритм выделяет группы похожих временных срезов в пространстве macrostate-признаков. После этого каждый кластер интерпретируется по средним значениям признаков и получает содержательное название, например `Regular Structure / Micelle` или `Sparse Distribution`. Для multistep и synthetic rollout используется уже обученная модель KMeans: новые временные срезы не переобучают модель, а только получают `cluster_id` ближайшего центроида.

На каждом временном срезе система роботов агрегируется в вектор из 11 macrostate-признаков:

```text
Polar_Order
Mean_Angle
Coordination_Num
Mean_Distance
Angular_Vel
Rotation_Direction
Rot_Order
Center_Vel
Velocity_Dispersion
Std_Nearest_Dist
Mean_Velocity
```

Эти признаки описывают несколько аспектов коллективной динамики. `Polar_Order` и `Rot_Order` характеризуют степень согласованности ориентаций и вращательного движения. `Coordination_Num`, `Mean_Distance` и `Std_Nearest_Dist` отражают плотность, локальную структуру и характерные расстояния между роботами. `Angular_Vel`, `Rotation_Direction`, `Center_Vel` и `Mean_Velocity` описывают динамический режим: вращение, смещение центра масс и общий уровень активности. `Velocity_Dispersion` показывает, насколько согласованно или хаотично движутся роботы.

Кластеризация выполняется методом KMeans с числом кластеров:

```text
n_clusters = 4
```

Перед KMeans признаки стандартизируются через `StandardScaler`, чтобы признаки с разными масштабами не доминировали в евклидовом расстоянии до центроидов. Модель обучается только на фактических экспериментальных данных. Такой дизайн важен методологически: фактические траектории задают эмпирическое пространство возможных макросостояний, а прогнозные и синтетические траектории затем соотносятся с уже найденными состояниями.

В результате были выделены четыре типа макросостояний. Основные интерпретируемые состояния включают:

```text
Regular Structure / Micelle
Sparse Distribution
Intermediate / transition state
Rotational / swarm-like state
```

Интерпретация кластеров выполняется по средним значениям macrostate-признаков внутри каждого кластера. Например, состояние `Regular Structure / Micelle` соответствует более плотной и структурированной конфигурации роботов, а `Sparse Distribution` — разреженному состоянию с увеличенными расстояниями между роботами. Переходные и swarm-like состояния отражают промежуточные режимы, в которых система ещё не сформировала устойчивую мицеллоподобную структуру или находится в более динамичном коллективном режиме.

Ключевой результат этого этапа — построен единый интерфейс между микродинамикой и макродинамикой:

```text
координаты роботов → macrostate features → KMeans cluster_id → интерпретируемое макросостояние
```

Это позволило перейти от вопроса «насколько точно предсказаны координаты?» к более содержательному вопросу: «какое коллективное состояние принимает система и как оно меняется во времени?». В synthetic macrostate study каждый из 72 синтетических rollout длиной 600 шагов получает последовательность меток кластеров, после чего вычисляются метрики появления и устойчивости мицеллоподобного состояния:

```text
micelle_formed
first_micelle_step
micelle_total_steps
micelle_max_lifetime_steps
final_state
```

Таким образом, Macrostate KMeans-модель является связующим компонентом между GNN-прогнозом и исследовательским выводом о влиянии начальной плотности, скорости и режима движения на формирование мицеллы.

### 6. Synthetic Macrostate Study

Для проверки влияния начальных условий реализован программный генератор синтетических начальных состояний. Он создаёт стартовую историю системы, достаточную для расчёта скоростей и ускорений, после чего GNN выполняет autoregressive rollout на 600 шагов. В последней серии экспериментов использовалась следующая сетка:

```text
n_robots = 50
total_steps = 600
dt = 0.1
center = (1000, 500)
radius_values = [250, 350, 470]
speed_values = [1.0, 3.0, 6.0]
velocity_modes = [random, aligned, rotational, inward]
repeats = 2
```

Всего было проведено:

```text
3 radii × 3 speeds × 4 velocity modes × 2 repeats = 72 synthetic experiments
```

Для каждого эксперимента сохраняются координаты rollout в формате фактических данных:

```text
file_name, slice_id, bot_id, coord_x, coord_y, angle
```

Это позволяет повторно применять обученную KMeans-модель и строить визуализации так же, как для фактических экспериментов.

## Полученные результаты

### 1. Данные и EDA

В основном наборе данных было проанализировано:

- **307 экспериментов**;
- **8 185 230 записей**;
- средняя длина эксперимента — **591.5 временных срезов**;
- максимальная длина — **888 срезов**;
- минимальная длина — **288 срезов**.

В ходе EDA было показано, что средняя скорость роботов не превышает **10 условных единиц** в **99%** экспериментов основного набора. Размер роботов оценён примерно как **50–60 условных единиц**, что использовалось при выборе локального графового взаимодействия и параметров синтетической генерации. Также было выявлено выраженное преимущественное направление вращения в части экспериментов: средняя доля роботов, вращающихся против часовой стрелки, составляла **0.839**, а в **270 экспериментах** наблюдалось преобладание направления более **70%**.

### 2. One-step прогноз: GNN против кинематического baseline

Для one-step прогноза была проведена оценка GNN относительно кинематического baseline. GNN показала существенное улучшение качества по всем основным компонентам:

| Метрика | Kinematic baseline | GNN | Улучшение |
|---|---:|---:|---:|
| RMSE `Δx` | 4.6028 | 2.4088 | +47.7% |
| RMSE `Δy` | 4.2510 | 2.3321 | +45.1% |
| RMSE `Δangle` | 19.1541 | 4.0699 | +78.8% |
| MAPE `Δx` | 5318.9136 | 2248.4402 | +57.7% |
| MAPE `Δy` | 5205.4033 | 2038.5692 | +60.8% |
| MAPE `Δangle` | 16464.2832 | 1929.8191 | +88.3% |

Наиболее сильный эффект наблюдается по углу ориентации: ошибка `Δangle` снижена почти на **79%** по RMSE. Это показывает, что GNN лучше учитывает коллективное взаимодействие роботов, чем чистая кинематическая экстраполяция.

MAPE для дельт имеет высокие абсолютные значения из-за того, что реальные `Δx`, `Δy`, `Δangle` часто близки к нулю. Поэтому основной интерпретируемой метрикой качества является RMSE.

### 3. Macrostate-кластеризация

Была реализована KMeans-кластеризация с **4 кластерами** на основе 11 macrostate-признаков. Модель KMeans обучалась на фактических данных и далее использовался без переобучения для анализа synthetic rollout. Это позволило перейти от координатного уровня к состояниям системы.

Практический результат этого этапа — возможность для каждого временного среза автоматически получать:

```text
cluster_id
state_name
micelle flag
```

Такой подход позволил вычислять не только координатные ошибки, но и макродинамические показатели: появилась ли мицелла, на каком шаге она появилась, сколько шагов существовала и в каком состоянии система оказалась в конце rollout.

### 4. Synthetic Macrostate Study

В synthetic study было проведено **72 эксперимента**:

```text
3 initial radius × 3 speed values × 4 velocity modes × 2 repeats
```

Каждый эксперимент имел длину **600 шагов**. На каждом шаге trajectory rollout получал метку ближайшего кластера по обученной KMeans-модели.

Общие результаты последней серии synthetic simulation:

- всего экспериментов: **72**;
- мицелла появилась хотя бы один раз: **68 / 72**, то есть **94.4%**;
- финальное состояние `Regular Structure / Micelle`: **23 / 72**, то есть **31.9%**;
- финальное состояние `Sparse Distribution`: **49 / 72**, то есть **68.1%**.

Важно, что `micelle_formed` и `final_state` — разные показатели. `micelle_formed=True` означает, что мицелла возникла хотя бы один раз за 600 шагов. `final_state` показывает только состояние на последнем шаге. Поэтому возможна ситуация, когда мицелла сформировалась в начале или середине rollout, но к 600-му шагу система перешла в разреженное состояние.

### 5. Влияние начальной плотности

Начальный радиус области размещения роботов оказался ключевым фактором формирования мицеллы:

- при `radius = 250` мицелла появлялась в **100%** экспериментов;
- при `radius = 350` мицелла появлялась в **100%** экспериментов;
- при `radius = 470` вероятность снижалась примерно до **83%**.

Это показывает, что плотные и среднеплотные начальные конфигурации почти гарантированно приводят к формированию мицеллоподобного состояния, а разреженные конфигурации становятся чувствительнее к режиму начального движения.

### 6. Влияние начальной скорости

В synthetic study использовались три значения средней начальной скорости:

```text
speed_mean = 1.0, 3.0, 6.0 ед/шаг
```

Было показано, что увеличение скорости ускоряет появление мицеллы:

- при `speed = 1.0` средний первый шаг появления мицеллы — около **25.6**;
- при `speed = 3.0` — около **13.4**;
- при `speed = 6.0` — около **6.9**.

При этом более высокая скорость также может повышать длительность непрерывного существования мицеллы, но этот эффект зависит от начального радиуса и режима движения.

### 7. Влияние режима начального движения

Сравнивались четыре режима начального движения:

```text
random, aligned, rotational, inward
```

По результатам synthetic study:

- `inward` формирует мицеллу быстрее всего: средний первый шаг около **6.2**;
- `aligned` также приводит к устойчивому появлению мицеллы, средний первый шаг около **11.1**;
- `random` имеет вероятность формирования около **0.89** и средний первый шаг около **10.1**;
- `rotational` имеет вероятность около **0.89**, но формирует мицеллу заметно позже — около **36.8** шага.

По средней максимальной непрерывной длительности мицеллы:

- `inward` — около **337 шагов**;
- `rotational` — около **264 шагов**;
- `random` — около **246 шагов**;
- `aligned` — около **209 шагов**.

Таким образом, направленное движение к центру является наиболее благоприятным режимом как для быстрого формирования, так и для более длительного существования мицеллы.

## Дальнейшие планы по развитию

Проект может быть развит в полноценную магистерскую и далее аспирантскую исследовательскую задачу. Основные направления развития следующие.

### 1. Улучшение физической корректности multistep прогноза

Текущий multistep rollout накапливает ошибку, поскольку после начальной истории модель использует собственные прогнозы. Следующий этап — разработка более устойчивых моделей долгосрочного прогноза:

- обучение GNN не только на one-step, но и на multi-horizon loss;
- scheduled sampling для уменьшения разрыва между train и inference;
- добавление физических ограничений: границы арены, ограничение скорости, запрет пересечения роботов;
- использование residual-correction схемы поверх кинематического baseline.

### 2. Переход к более инвариантным признакам

Текущая GNN использует абсолютные координаты, поэтому synthetic rollout должен быть задан в той же системе координат, что и fact-data. Для повышения переносимости модели планируется:

- нормировать координаты относительно центра арены;
- добавить boundary-aware признаки;
- заменить часть абсолютных признаков относительными;
- исследовать E(n)-equivariant или SE(2)-equivariant GNN-архитектуры.

Это позволит использовать модель для разных геометрий арены и разных начальных расположений без потери качества.

### 3. Развитие Macrostate-модели

KMeans является удобной и интерпретируемой первой моделью макросостояний, но дальше можно перейти к более выразительным методам:

- Gaussian Mixture Models для вероятностного описания кластеров состояний;
- HMM или switching state-space models для явного моделирования переходов между состояниями;
- contrastive/self-supervised learning для обучения latent macrostate representation;
- supervised-модель присвоения макросостояний при наличии экспертной разметки.

Отдельное направление — валидация названий кластеров через экспертную оценку и сопоставление с физическими характеристиками системы.

### 4. Расширение synthetic study

Текущая сетка включает 72 эксперимента. Для более статистически устойчивых выводов необходимо:

- увеличить число повторов с `2` до `10–30` на каждую конфигурацию;
- расширить диапазон радиусов, скоростей и числа роботов;
- добавить новые режимы начального движения;
- исследовать чувствительность к шуму, плотности и параметрам роботов;
- сравнить synthetic rollout с дополнительными физическими экспериментами.

Это позволит получить более строгие оценки вероятности формирования мицеллы и построить карту фазовых переходов системы.

### 5. Подготовка к научной публикации

Работа может быть развита в публикацию по теме data-driven modeling of robotic swarms. Возможные научные вклады:

- воспроизводимый pipeline для перехода от микродинамики к макросостояниям;
- сравнение кинематического baseline и GNN для one-step прогноза;
- synthetic macrostate study для анализа влияния начальных условий;
- методология кластеризации macrostate-признаков на основе фактических данных и применения к прогнозным rollout.

В перспективе проект может быть расширен до магистерской выпускной работы, а затем до аспирантского исследования по моделированию, управлению и интерпретации самоорганизующихся multi-agent robotic systems.
